# Downstream ideas after Research Memory OS

## 1. Research cockpit

A landing page that answers:

- What do I currently care about?
- What changed this week?
- Which knowledge nodes grew?
- Which claims are unsupported?
- Which papers should I read next?
- Which assumptions should I revisit?

## 2. Literature review generator with trace

Generate a review section from knowledge graph communities rather than from raw notes. Every paragraph should include:

- supporting papers
- evidence spans
- open conflicts
- confidence level
- missing citations

## 3. Hypothesis workbench

Turn metacognitive insights into tracked hypotheses:

- supporting evidence
- counter evidence
- experiments needed
- related datasets
- expected failure cases
- next paper search queries

## 4. Research direction drift detector

Compare recent events and interests against declared profiles:

- detect new hidden interests
- warn when a profile is stale
- suggest profile split/merge
- show why radar recommendations are drifting

## 5. Claim conflict map

Use `knowledge_edges` and `atomic_knowledge_units` to show:

- claims that contradict each other
- papers with incompatible assumptions
- datasets where conclusions diverge
- methods that only work under narrow conditions

## 6. “What should I read next?” recommender

Rank papers by expected memory update value rather than relevance alone:

- will it fill a gap?
- will it resolve a conflict?
- will it update a high-value method node?
- will it test a current hypothesis?

## 7. Methodology coach

Use `methodology_rules` to guide reading and research design:

- per-field reading templates
- experiment design checklist
- baseline and metric checklist
- failure mode checklist
- paper critique template

## 8. Memory debugger for every answer

For each answer or recommendation, show:

- source evidence
- semantic nodes used
- episodic signals used
- methodology rules used
- metacognitive assumptions used
- missing or weak evidence

## 9. Knowledge tree version diff

Compare taxonomy versions:

- added/removed nodes
- merged/split concepts
- changed parent-child links
- newly dominant method families
- topics whose confidence decreased

## 10. Personal research autopilot

A weekly system that proposes:

- papers to deep-read
- profiles to edit
- obsolete interests to archive
- hypotheses to test
- claims needing evidence
- experiments or notes to write

Keep this as a proposal system, not an automatic decision maker. High-risk memory writes should still go through review.
